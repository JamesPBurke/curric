Switch Project Guide

Student-facing guide for building a Micro:bit switch mini-project.

Course: Engineering Design

Project Length: About 2–3 weeks

Project Goal: Build a working physical switch that connects to a Micro:bit and triggers a programmed output.

Project Summary

In this project, you will create a physical switch from simple materials such as cardboard and aluminum foil. Your switch will connect to the Micro:bit through its pins. When the switch closes the circuit, your code should detect the change and produce an output.

This project introduces physical computing: code that responds to something happening in the real world.

Learning Targets

  • identify the Micro:bit pins used in a simple circuit
  • explain how a switch opens and closes a circuit
  • connect a physical input to a programmed output
  • test and improve a prototype for reliability

Success Criteria

  • the switch reliably changes the state of the circuit
  • the Micro:bit responds correctly when the switch is used
  • the wiring is safe and organized
  • you can explain how the switch works
  • you improve the design based on testing

Suggested Materials

  • 1 Micro:bit
  • battery pack or USB connection
  • alligator clips / jumper leads
  • cardboard
  • aluminum foil
  • tape or glue
  • scissors
  • optional: binder clips, paper fasteners, clothespins, or extra cardboard supports
  • optional: LED and resistor for external output

Background Knowledge

Important pins

  • P0 / P1 / P2: can be used for input or output
  • GND: ground, used to complete the circuit
  • 3V: power pin

Key idea

A switch changes whether a circuit is open or closed.

  • Open circuit = no complete path
  • Closed circuit = complete path, so the signal can be detected

Build Challenge

Design a switch that:

  • can be pressed, squeezed, folded, or otherwise activated by a user
  • changes the signal going into the Micro:bit
  • is stable enough to work more than once
  • does not accidentally stay shorted all the time

Step-by-Step Process

Part 1: Plan

  1. Sketch your switch idea.
  2. Label the two parts that will make contact.
  3. Decide how the switch will return to its open state.
  4. Predict what could go wrong.

Part 2: Build

  1. Cut cardboard pieces for the base and moving part.
  2. Attach foil where the conductive surfaces should touch.
  3. Tape or glue the structure together.
  4. Leave room for clips or wires to connect.
  5. Connect one side to a signal pin and one side to ground.

Part 3: Code

Create a MakeCode program that does something when the switch is activated.

Possible responses:

  • show an icon
  • clear the screen when not pressed
  • turn on an LED
  • flash a pattern

Part 4: Test

Run multiple trials.

Ask:

  • Does it trigger every time?
  • Does it trigger when it should not?
  • Does the mechanism stay lined up?
  • Do the wires stay connected?

Part 5: Improve

Revise your design to make it more reliable, easier to use, stronger, and clearer to understand.

Troubleshooting Questions

  • Is the code correct?
  • Are the wires connected to the intended pins?
  • Is one side connected to ground?
  • Are the foil surfaces actually touching?
  • Are they touching too much and creating a permanent short?
  • Is the structure too loose or unstable?

Reflection Questions

  1. What design did you build, and how does it work?
  2. What problem did you encounter most often?
  3. What change improved reliability the most?
  4. If you rebuilt it, what would you change?
