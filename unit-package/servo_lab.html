Servo Lab

Safe wiring and basic motion practice for the Micro:bit servo lab.

Course: Engineering Design

Lab Length: 1–2 class periods

Purpose: Learn how to safely connect and test a servo before integrating it into the foam car project.

Lab Question

How can we safely control motion with a Micro:bit and a servo motor?

Learning Targets

  • identify the difference between continuous-rotation and positional servos
  • safely wire a servo for testing
  • write simple code to control servo behavior
  • explain why motor power and control wiring must be handled carefully

Important Safety Note

In classroom builds, do not assume the servo should be powered directly from the Micro:bit. A separate battery pack for the servo system is the safer default.

The Micro:bit can send the control signal, but the motor system may need its own power source.

Critical rule: If you use separate power for the servo, the servo system and the Micro:bit still need a common ground.

Servo Types

1. Positional servo

  • moves to a requested angle
  • often used for steering or controlled arm movement
  • example: blue servo in some classroom kits

2. Continuous-rotation servo

  • spins forward or backward instead of moving to a fixed angle
  • useful for wheels and drivetrain motion
  • often used as the main drive motor in simple robot/car builds
  • example: yellow servo in many classroom kits

Materials

  • 1 Micro:bit
  • programming cable
  • battery pack for Micro:bit
  • 1 servo motor
  • separate servo battery supply if required
  • wires / clip leads / breadboard / breakout board as needed
  • reference diagram from teacher

Wiring Concepts

A servo usually has three wires: signal, power, and ground.

Common color pattern:

  • orange/yellow = signal
  • red = power
  • brown/black = ground

Standard concept for this lab:

  • signal wire goes to a Micro:bit pin, often P0
  • ground must connect correctly
  • power must come from the approved source for your classroom setup

Do not invent your own power arrangement unless instructed to do so.

Procedure

Part 1: Identify the servo type

  1. Look at the servo label or teacher reference.
  2. Decide whether it is positional or continuous rotation.
  3. Record how you know.

Part 2: Wire the system

  1. Connect the servo signal line to the assigned Micro:bit pin.
  2. Connect grounds correctly.
  3. Connect the servo power using the approved setup.
  4. Have the wiring checked before powering on.

Part 3: Write a simple test program

Create a MakeCode program that sends a control signal to the servo.

If it is a positional servo: test several angles such as center, left or low angle, right or high angle.

If it is a continuous-rotation servo: test for stop / neutral, forward motion, reverse motion.

Troubleshooting Guide

  • Is the signal wire connected to the correct pin?
  • Is the code using the same pin as the wiring?
  • Is the servo getting approved power?
  • Is the ground shared correctly?
  • Is the servo type the one you think it is?
  • Is the neutral value for a continuous servo slightly off and in need of calibration?

Reflection Questions

  1. Which type of servo did you use?
  2. What was the most difficult part of the setup?
  3. What values or commands produced useful behavior?
  4. Why is a separate motor power source often safer than powering directly from the Micro:bit?
  5. How will this lab help with the foam car build?

Exit Ticket

In 3–5 sentences, explain how your servo was wired, what your test code did, and what you learned that will matter for the car project.
